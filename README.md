# AI-агенты (Нейросотрудники)

Мобильное веб-приложение для общения с ИИ-агентами («нейросотрудниками»).  

---

## Функционал

- Авторизация (имитация, тестовый логин/пароль)
  
- Главное меню с разделами:
  - Мои нейросотрудники
  - Готовые нейросотрудники
  - Настройки

- Списки сотрудников с карточками:
  - аватар (иконка или первая буква имени)
  - имя, описание, статус
  - индикатор количества новых сообщений

- Чат с каждым сотрудником:
  - отправка текстовых сообщений
  - индикатор «печатает...» (задержка 1.5 сек)
  - сохранение истории в localStorage

- Прикрепление файлов (изображения, PDF, документы, аудио):
  - отображается имя, тип, размер, иконка
  
- Настройки:
  - информация о текущем пользователе
  - очистка локального кэша (сохраняется авторизация)
  - переключение светлой / тёмной темы
  - выход из профиля
  
- Адаптивный дизайн (мобильная версия)

- Переключение темы с сохранением в localStorage

---

## Стек технологий

| Компонент | Технологии |
|-----------|------------|
| Бэкенд | Django, Django REST Framework, CORS |

## Архитектура 

1. Бэкенд (Django + DRF)
   
    -Отдаёт мок-данные по API-эндпоинтам.

    -Не использует базу данных (все данные — статические).

    -Настроен CORS для взаимодействия с фронтендом.

2. Фронтенд (TypeScript + HTML/CSS)
   
    -Одностраничное приложение (SPA) с динамической подгрузкой шаблонов.

    -Каждая страница — отдельный класс, который рендерит <template> из mobile.html.

    -api_client.ts — отдельный слой для работы с API (легко заменить на реальный бэкенд).

    -storage.ts — работа с localStorage (сохранение пользователя, сообщений, темы).

    -Страницы не зависят друг от друга, навигация через MenuPage.

## Имитация (мок-данные)

-Авторизация: работает только с тестовой парой test@example.com / 123456.

-Списки сотрудников: данные зашиты в бэкенде (MY_EMPLOYEES, READY_EMPLOYEES).

-Ответы бота в чате: фиксированный набор фраз, зависящий от ключевых слов.

-Отправка файлов: файлы не отправляются на сервер, только отображаются локально.

-База данных: отсутствует (бэкенд не использует БД).


---

## Что нужно заменить для подключения реального API

-api_client.ts — заменить BASE_URL на реальный адрес, обновить функции под формат реального бэкенда.

-Бэкенд: полностью заменить мок-данные на реальные эндпоинты (аутентификация, получение сотрудников, отправка сообщений, загрузка файлов и т.д.).

-Авторизация: убрать имитацию, подключить реальный механизм (токены / сессии).

-Сохранение данных: если реальный API требует отправки файлов или истории сообщений — добавить соответствующие вызовы.
